Python - Multiprocessing

Multiprocessing is a Python package that supports spawning processes using an API similar to the threading module [1].

In general, a process is the execution of a program whereas a thread is the execution unit part of a process [2]. Whereas a thread is lightweight, a process is heavyweight - processes do not share data and memory with one another while threads do.

A process executes a program; you can have multiple processes executing the same program, but each process has its own copy of the program within its own address space and executes it independently of the other copies." [2]

Example:

#!/usr/bin/env python
import os
import multiprocessing

def child_process():
    print(f"Hi! I'm a child process {os.getpid()}")

if __name__ == "__main__":
    print(f"Hi! I'm process {os.getpid()}")

    # Here we create a new instance of the Process class and assign our
    # `child_process` function to be executed.
    process = multiprocessing.Process(target=child_process)

    # We then start the process
    process.start()

    # And finally, we join the process. This will make our script to hang and
    # wait until the child process is done.
    process.join()

    print("AFTER CHILD EXECUTION! RIGHT?!")

Remember to use .join() to wait for your child process to execute and finish. Use .join() wisely as it will determine how you want the module to perform. The output above will be:

Hi! I'm process 4172
Hi! I'm a child process 5372
AFTER CHILD EXECUTION! RIGHT?!

However, had we swapped the last line before process.join():

    process.start()

Communication between child and parent process Link to heading

How do you return data from a child process? One would think to use the return statement but in processes, it is not allowed because of isolation. Instead, we use the queue (FIFO, first-in-first-out) class, which will provide an interface to communicate data between the parent and child process. ‘Nuff talking, let’s see an example:

#!/usr/bin/env python
import os
import multiprocessing

def child_process(queue, number1, number2):
    print(f"Hi! I'm a child process {os.getpid()}. I do calculations.")
    sum = number1 + number2

    # Putting data into the queue
    queue.put(sum)

if __name__ == "__main__":
    print(f"Hi! I'm process {os.getpid()}")

    # Defining a new Queue()
    queue = multiprocessing.Queue()

    # Here we create a new instance of the Process class and assign our
    # `child_process` function to be executed. Note the difference now that
    # we are using the `args` parameter now, this means that we can pass
    # down parameters to the function being executed as a child process.
    process = multiprocessing.Process(target=child_process, args=(queue,1, 2))

    # We then start the process
    process.start()

    # And finally, we join the process. This will make our script to hang and
    # wait until the child process is done.
    process.join()

    # Accessing the result from the queue.
    print(f"Got the result from child process as {queue.get()}")

Exception handling for the process class Link to heading

Exceptions that occurs inside a child process will always be handled by the Process class that spawned it [1].

#!/usr/bin/env python
import os
import multiprocessing

def child_process():
    print(f"Hi! I'm a child process {os.getpid()}.")
    raise Exception("Oh no! :(")

if __name__ == "__main__":
    print(f"Hi! I'm process {os.getpid()}")

    # Here we create a new instance of the Process class and assign our
    # `child_process` function to be executed. Note the difference now that
    # we are using the `args` parameter now, this means that we can pass
    # down parameters to the function being executed as a child process.
    process = multiprocessing.Process(target=child_process)

    try:
        # We then start the process
        process.start()

        # And finally, we join the process. This will make our script to hang and
        # wait until the child process is done.
        process.join()

        print("AFTER CHILD EXECUTION! RIGHT?!")
    except Exception:
        print("Uhhh... It failed?")

This tells us that the parent process is still running even though the child process threw an exception. Instead, we should handle the exceptions inside the child process:

#!/usr/bin/env python
import os
import multiprocessing

def child_process():
    try:
        print(f"Hi! I'm a child process {os.getpid()}.")
        raise Exception("Oh no! :(")
    except Exception:
        print("Uh, I think it's fine now...")

if __name__ == "__main__":
    print(f"Hi! I'm process {os.getpid()}")

    # Here we create a new instance of the Process class and assign our
    # `child_process` function to be executed. Note the difference now that
    # we are using the `args` parameter now, this means that we can pass
    # down parameters to the function being executed as a child process.
    process = multiprocessing.Process(target=child_process)

    # We then start the process
    process.start()

    # And finally, we join the process. This will make our script to hang and
    # wait until the child process is done.
    process.join()

    print("AFTER CHILD EXECUTION! RIGHT?!")

And the output:

Hi! I'm process 28124
Hi! I'm a child process 2824.
Uh, I think it's fine now...
AFTER CHILD EXECUTION! RIGHT?!

Key takeaway points when using Multiprocessing package:

  • start/join: for every process start, keep track and then join the process
  • use queue for child-parent data/return communications
  • handle exceptions within the process itself as they run independently
  • use for-loops to distribute loads and track with args
