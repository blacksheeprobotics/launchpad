QNX RPi Development Workflow

February 23, 2025 3-minute read
QNX • RPi

Source: QNX Toolkit for Visual Studio Code + QNX 8.0 Deep Dive

If you need more info about the image, see the WIKI

This section is quick summary for myself. Watch the video if you’ve never done this before.

  1. Open up VSCode and click on the QNX extension module in the menu bar
  2. In QNX COMMANDS > Projects, select Create new Project
  3. Enter the project name in the field
  4. In select project template, choose QNX Recursive Makefile
    • introduces a recursive directory tree
    • allows you to compile different binaries for different architectures
  5. In select language, choose c or cpp
  6. In Select artifact type of project to build: choose Executable
  7. Write your code
  8. To build, right click on the project name in the explorer
  9. Scroll down to the QNX tab, hover and select Build Active Project
  10. To run the code on the QNX device, right click project name in explorer. Scroll down to the QNX tab and select Run As QNX Application
  11. If you haven’t chosen a target yet, QNX will ask you for a target
  12. Click the QNX extension module and create the target in QNX TARGETS
  13. For the RPi, get its IP address on the network (like Angry IP Scanner or in serial terminal, use ifconfig)
  14. Get the address and return to VS Code and enter it to Add new QNX Target Connection in QNX TARGETS
  15. To access target via SSH, use:
> ssh -m hmac-sha2-512 qnxuser@qnxpi.local

OR

> ssh -m hmac-sha2-512 root@qnxpi.local
  1. Go back to your project in explorer, right click and go to QNX, Run As QNX Application.

How to DEBUG the Hello World Application on Raspberry Pi QNX Target Link to heading

  1. Right click on Hello_World and go to QNX > Debug as QNX Application
  2. Note: the toolkit will port a copy of the executable to /tmp

WORKFLOW Link to heading

  1. Lets say now we have a working c program in the Raspberry Pi
  2. We make some changes, say we add printf("I'm a new line!")
  3. At this point, we’ve made changes and we now need to do the following:
  4. Compile/Build it via the commands/instruction above
  5. Then Run As QNX Application or go to /tmp
  6. We can also access parameters about the system
    • use pidin info
    • what processes pidin arg

COMMAND-LINE WORKFLOW Link to heading

  1. Change directory to your QNX install directory
  2. cd qnx800
  3. If you list the files ls, you’ll see qnxsdp-env.sh
  4. Run it source qnxsdp-env.sh
  5. This will create the environment variables on your machine to allow CL commands and execute
  6. Go to your development directory. For now, go back into your hello_world directory:
  7. cd ../qnxprojects/hello_world/
  8. tree // works with linux only, not Windows
  9. make clean to clean the make files earlier
  10. use make again to regenerate the binaries
  11. After we build our program, to transfer the program/executable over to the remote QNX machine
  12. scp nto/aarch64/o-le/hello_world root@<IP address or hostname.local>:/tmp
  13. If Windows, use the x86_64 directory:
  14. scp nto/x86_64/o/hello_world root@<IP address or hostname.local>:/tmp
